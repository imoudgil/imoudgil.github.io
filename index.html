NeuroSync

C++20 · SITL motion engine
C++20 POSIX Threads Unix Sockets Quintic Splines

Producer–consumer engine bridging a slow Python policy (~1 Hz) to a 1000 Hz real-time control loop via IPC and trajectory interpolation.

  • TrajectoryBuffer (deque + mutex + condition_variable); consumer uses non-blocking tryPop() to keep RT loop off the wait path
  • Closed-form quintic spline solve for C²-continuous position/velocity/acceleration; sleep_until on steady_clock for 1 ms ticks
  • Newline-framed JSON over AF_UNIX with partial-read reassembly — zero third-party deps
